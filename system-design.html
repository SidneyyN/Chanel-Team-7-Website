SYSTEM DESIGN

System Architecture

This section introduces the design of the mobile application. We have two system archiecture diagrams for our situation. The first one is the 'ideal' situation, which is how it will look like when the app is deployed in boutiques. The second one is the current situation, which is how the application looks at the moment.

Y2-Point-Of-Sale-System (POS System) keeps track of products yet to be sold. However once an item has been purchased, the product exits this system and therefore cannot be tracked anymore. When the app is deployed and used alongside the POS System, products that exits the system can be added to the SQL Database.

The SQL Database stores items that are currently in boutique drawers to be shipped. It keeps tracks of which drawer each product is in and other relevant details.

Our Microsoft Powerapp Application will then take in data from the SQL database, and let fashion advisors edit orders and update the status. It is then linked to the Address Lookup API, which can store the address of customers using their post codes.

At the current moment, Microsoft Sharepoint List is functionally acting as our SQL database, as it stores order details and items in each drawers. It has the same role as a database.
