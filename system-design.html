SYSTEM DESIGN

System Architecture

This section introduces the design of the mobile application.

The first diagram represents the architecture of our app. The client interacts with our internal MS PowerApps application, which sends requests to the connected MS Sharepoint Lists. It acts as the application’s database, retrieving or updating its records, according to request received. Subsequently, the application processes the response and displays the relevant information back to the client, ensuring a seamless flow of information and interaction within the application.

Here is a second diagram which illustrates our envisioned ideal architecture for the application. The additional connections would improve scalability, efficiency and convenience of the application. It includes a connection to a proper database for scalability, and a connection to Chanel’s Y2 Point-of-Sale system. This would allow data about sold items that need to be shipped to transfer automatically to the PowerApp, making it more convenient and efficient. Additional time and permissions to access Chanel's resources would allow for further enhancements and progress of the app towards our ideal system architecture diagram.

Data Storage and Entity Relations

MS Sharepoint lists imitates a SQL database for our application. Tables are represented with seperate lists. Like a SQL database, lists store records with one or more attributes. To represent relations, SharePoint lists have 'lookup columns' to track and maintain information across multiple lists.
